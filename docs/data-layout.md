# Data layout

The portal's large datasets are kept **out of git** and provided separately. This
document describes what the code expects and how to point it at your store.

## Directories

All paths are configurable via `.env` (read by `services/api/config.py`); the
defaults below are relative to the repo root.

| Env var | Default | Contents | Origin |
|---------|---------|----------|--------|
| `TUNDRA_FOOTPRINTS` | `map/Yamal-Nenets/map_footprints_wgs84.gpkg` | datum-corrected sheet footprints | generated by `utils.qa_report` |
| — | `map/Yamal-Nenets/<pack>/maps/*.map,*.gif` | raw OziExplorer sheets | source archive |
| `TUNDRA_HISTORIC_COG` | `data/historic_cog/Yamal-Nenets/*.tif` | warped historic COGs | generated by `utils.warp` |
| `TUNDRA_CACHE` | `data/cache/composites/` | API satellite composite cache | generated at runtime |
| `TUNDRA_SAM_DIR` | `data/models/sam/` | SAM weights (`sam_vit_h_*.pth`, ~2.5 GB) | auto-downloaded on first segmentation |
| `TUNDRA_DATA` | `data/` | parent of the above `data/` subdirs | your store |

## Providing the data

`map/` and `data/` are gitignored. On any machine, provide them as real
directories or symlinks to your store. Example (Linux server):

```bash
# historic + modern rasters, model cache live on a big volume:
ln -s /srv/tundra-data data
# historic map sources (or copy them in):
ln -s /srv/tundra-maps map
```

Or set explicit absolute paths in `.env`:

```
TUNDRA_DATA=/srv/tundra-data
TUNDRA_FOOTPRINTS=/srv/tundra-maps/Yamal-Nenets/map_footprints_wgs84.gpkg
TUNDRA_HISTORIC_COG=/srv/tundra-data/historic_cog/Yamal-Nenets
```

## Sizes (Yamal-Nenets, for planning transfers)

| Dataset | Approx size |
|---------|-------------|
| Historic COGs (871 sheets) | ~36 GB |
| Raw `.map` + `.gif` sources | ~a few GB |
| SAM `vit_h` weights | ~2.5 GB (auto-download; no need to transfer) |
| Footprints GPKG | < 5 MB (regenerable) |
| Satellite composites | on-demand / cached; grows with use |

## Regenerating derived data

Everything except the raw sources is reproducible:

```bash
python -m utils.qa_report --base map/Yamal-Nenets --export          # footprints
python -m utils.warp --batch --base map/Yamal-Nenets \
    --out data/historic_cog/Yamal-Nenets --jobs 8                    # COGs (needs gdal-bin)
```

Satellite composites and SAM weights are produced/fetched automatically by the API.
