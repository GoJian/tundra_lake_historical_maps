# Tundra Historic Map Portal — Python runtime dependencies.

# Install into a virtualenv:
#     python3.12 -m venv .venv && source .venv/bin/activate
#     pip install --upgrade pip
#     pip install -r requirements.txt
#
# NOTES
#  * rasterio / fiona / pyproj ship manylinux wheels that BUNDLE GDAL/GEOS/PROJ,
#    so a plain pip venv works for the API and processing code — no system GDAL
#    needed at runtime.
#  * EXCEPTION: utils/warp.py (one-time historic GIF -> COG preprocessing) shells
#    out to the GDAL command-line tools. Those are NOT in the wheels; install
#    system GDAL for that step only:   sudo apt-get install -y gdal-bin
#  * torch: the default PyPI wheel is CUDA-enabled on Linux. For a specific CUDA
#    build or CPU-only, install torch separately first, e.g.:
#       pip install torch==2.7.1 --index-url https://download.pytorch.org/whl/cu124
#  * segment-geospatial pulls a broad dependency tree; only `segment_anything` +
#    `torch` are used by utils/segment.py. Kept as-is because it is the tested set.
